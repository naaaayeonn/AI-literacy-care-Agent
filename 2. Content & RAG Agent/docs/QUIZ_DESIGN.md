# 퀴즈 생성 설계 가이드 (QUIZ_DESIGN.md)

이 문서는 AI 리터러시 케어 시스템에서 사용되는 퀴즈 생성 엔진(`quiz_generator.py`)의 작동 원리, 프롬프트 템플릿, 유효성 검사 기준 및 Fallback 처리를 기술합니다.

---

## 1. 퀴즈 생성 원칙

사용자의 몰입감과 리터러시 진단을 위해 퀴즈는 다음 4가지 핵심 원칙을 따릅니다.

1. **철저한 문맥 근거성 (Context Grounding)**
   - 퀴즈는 반드시 주어진 청크 텍스트(`context`)의 내용만을 기반으로 생성되어야 합니다.
   - 외부 지식을 요구하거나 본문과 모순되는 정보는 배제합니다.
2. **명확한 정답성 (Single Clear Answer)**
   - 4지선다 형식 중 오직 1개만이 논란의 여지 없는 명확한 정답이어야 합니다.
3. **그럴듯한 오답 설계 (Plausible Distractors)**
   - 나머지 3개의 오답은 본문 단어들을 조합해 그럴듯하게 구성하되, 본문을 정확하게 이해한 독자는 확신을 갖고 제외할 수 있어야 합니다.
4. **인용 기반 정답 해설 (Quote-based Explanation)**
   - 정답 해설에는 본문 문장을 직접 인용하거나 유추적 단서를 근거로 기술해야 합니다.

## 2. 퀴즈 데이터 규격 (QuizDict)

API 통신 및 백엔드/프론트엔드 매핑 시 사용되는 JSON 규격은 다음과 같습니다.

```json
{
  "chunk_id": "chunk_doc001_03",
  "question": "본문에서 설명하는 메타인지의 가장 올바른 정의는 무엇인가요?",
  "options": [
    "1. 타인과 경쟁하여 학습 능력을 키우는 외적 동기",
    "2. 자신의 생각과 인지 과정을 스스로 성찰하고 조절하는 능력",
    "3. 컴퓨터의 연산 장치 중 중앙처리장치를 일컫는 한자어",
    "4. 한꺼번에 번역하여 컴파일하는 기계 프로그래밍 번역 기술"
  ],
  "correct_option": 2,
  "explanation": "본문에서 메타인지란 '자신의 사고 과정을 인식하고 조절하는 능력'이라고 정의하고 있으므로, 2번이 가장 적절합니다."
}
```

## 3. 프롬프트 구조 (prompts.py)

### System Prompt
```text
당신은 독해력 향상을 위한 인터랙티브 퀴즈를 만드는 전문가입니다.
... (생략) ...
출력 형식 (반드시 이 JSON만 출력):
{
  "question": "질문 내용",
  "options": ["1. 선택지1", "2. 선택지2", "3. 선택지3", "4. 선택지4"],
  "correct_option": 정답번호(1~4 중 하나의 정수),
  "explanation": "정답 해설"
}
```

### User Prompt
```text
다음 텍스트를 읽고 독해력 확인 퀴즈를 만들어 주세요.

[텍스트]
{chunk_text}

[퀴즈 JSON]
```

## 4. 유효성 검사 및 Fallback 로직

LLM이 생성한 퀴즈 결과물은 신뢰성이 보장되어야 하므로, 아래의 검증 단계를 거칩니다.

### 1) 유효성 검사 기준 (validate_quiz)
- `question` 필드가 문자열 타입이며 공백이 아닐 것.
- `options`가 정확히 4개의 문자열 아이템을 가진 리스트 타입일 것.
- `correct_option`이 정수형 타입이며 `[1, 2, 3, 4]` 범위 내에 있을 것.
- `explanation`이 문자열 타입이며 공백이 아닐 것.

### 2) Fallback 로직
유효성 검사에 실패하거나 LLM 서버 장애/네트워크 지연 발생 시, 프론트엔드가 크래시하는 것을 방지하기 위해 **미리 준비된 기본 독해력 퀴즈(Fallback Quiz)**를 즉시 반환합니다.

```python
# fallbacks.py
def fallback_quiz(chunk_id: str) -> QuizDict:
    return QuizDict(
        chunk_id=chunk_id,
        question="방금 읽은 문단의 핵심 내용으로 가장 적절한 것은 무엇인가요?",
        options=[
            "1. 본문에서 언급된 개념을 처음 제안한 사람이 누구인지 설명한다.",
            "2. 본문의 핵심 개념과 그 의미를 이해했다.",
            "3. 본문과 무관한 배경 지식이 필요하다.",
            "4. 본문의 내용은 다른 분야에도 동일하게 적용된다.",
        ],
        correct_option=2,
        explanation="이 퀴즈는 기본 독해력 확인 문항입니다. 본문의 핵심 개념 이해 여부를 확인하세요."
    )
```

## 5. Ragas 평가지표 연동 가이드 (5번 QA 협업)

퀴즈의 정답 신뢰도(Faithfulness)와 문맥 연관성(Context Relevance)은 5번 QA 평가 파이프라인과 연동하여 정밀 검증할 수 있습니다.
- **Ragas Question Harmful/Q-A Groundedness**: 생성된 질문과 해설이 실제 `context`에 얼마나 충실한지를 측정하여 평균 **0.85 이상**을 목표로 합니다.
