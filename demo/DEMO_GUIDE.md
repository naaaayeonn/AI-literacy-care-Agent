링크 하나(로컬 `http://localhost:8000`)에서 **1·2·3·4·5번 전체가 통합된 폐루프**가 돕니다:

## 2. 데모 당일 실행 절차

### ① 시작
`C:\team-full\demo\start_demo.bat` **더블클릭.**
- 검은 창에서 5단계(`[1/5]…[5/5]`)가 진행됩니다.
- 끝에 **`✅ 준비 완료!`** 가 뜨고 브라우저가 자동으로 `http://localhost:8000` 을 엽니다.
- 백엔드는 **별도 창**에서 계속 떠 있습니다(그 창은 닫지 마세요).
- 처음 실행 시 의존성 설치로 몇 분 걸릴 수 있습니다. **시연 전 미리 한 번 돌려두세요.**

### ② 점검 (권장)
브라우저가 뜬 뒤, 새 명령창에서:
```
cd C:\team-full
.\.venv\Scripts\python.exe demo\preflight.py
```
→ **`✅ 전체 통과 (8/8)`** 가 나오면 시연 준비 끝.
(8번 항목 `[ WARN ]` 은 정상입니다 — 사전 API 키 관련, 아래 4번 참고)

### ③ 데이터 초기화 (리허설 후)
여러 번 연습했다면, 브라우저 주소창에 아래를 한 번 실행해 기록을 깨끗이:
```
POST http://localhost:8000/api/session/reset
```
간단히는 시연 직전 `start_demo.bat`을 다시 돌리면 SQLite가 새로 만들어집니다.

### ④ 종료
`stop_demo.bat` 더블클릭.

## 3. 시연 대본 (멘트 포함 · 5~7분)

> 브라우저는 **전체화면(F11)** 권장.

**[1] 진입 — 익명 로그인 + 동의** (`/onboarding`)
> "저희 서비스는 로그인 없이 **익명 ID**로 시작합니다. 개인정보를 최소 수집하고 동의를 받은 뒤 시작하죠."
- 동의 체크 → **[익명으로 시작하기]**

**[2] 모드 선택** (`/home`)
> "준비된 글로 바로 체험하거나, 제가 읽고 싶은 문서를 올릴 수도 있습니다. 실제 브라우징은 확장으로도 확장되고요."
- **[실시간 케어 ON]** 클릭 (안정적인 데모 경로)

**[3] 읽기 + 실시간 개입** (`/reading`) — ★ 핵심
> "글을 읽는 동안 스크롤·체류·이탈로 **집중도**를 실시간 측정합니다. 집중이 떨어지면 개입하죠."
- 우측 **[집중도 시뮬] → `Hard(25)`** 버튼 클릭 → **집중도 급락 → 인터럽트 퀴즈 팝업**
> "이렇게 집중이 흐트러지면 방금 읽은 내용을 **퀴즈로 확인**시켜 능동적 이해를 유도합니다. GPT가 대신 읽어주는 것과 정반대죠."
- 퀴즈 정답 선택 → 점수 반영
- (선택) 본문의 **`환각`·`리터러시`** 같은 용어에 **마우스 올리면** RAG 용어 풀이 툴팁

**[4] 성장 추적** (`상단 [성장 대시보드]`) — ★ 심사 포인트
> "그리고 이 데이터로 **문해력 성장**을 추적합니다."
- **Literacy Score '케어 적용 vs 미적용' 전후 비교 그래프** 가리키며:
> "케어를 받은 경우(파란선)가 확연히 성장합니다. **측정·개입·추적**의 폐루프가 만든 결과입니다."
- 레이더(다면 역량) · 독해시간·XP 추이 · AI 코치 처방전 간단 소개

**[5] (여유되면) 내 문서 업로드**
> "준비된 글뿐 아니라, 실제 읽고 싶은 문서에도 그대로 적용됩니다."
- 상단 **[＋ 새 세션] → [페이지 업로드]** → 텍스트 붙여넣기 → **[읽기 시작]** → 내 글이 리더에 렌더

**마무리 한 줄**
> "GPT는 텍스트를 처리하지만, 저희는 **사람의 문해력 성장을 관리**합니다."

## 4. 시연 전 반드시 알아둘 점 (리스크)

- **집중도는 시뮬 버튼으로 유발하세요.** 실제 스크롤보다 `Hard(25)` 버튼이 100% 재현됩니다.
- **용어 조회 — 지금은 로컬 사전 단어만.** `환각·리터러시·LLM` 등 미리 담긴 용어는 **hover 툴팁**으로 확실히 뜹니다.
  임의 단어를 드래그하면 "🔍 뜻 보기"가 뜨지만, **사전 API 키(`WOORIMAL_API_KEY`/`STDICT_API_KEY`)가 서버 env에 설정돼야** 모든 단어가 풀립니다(현재 미설정 → 사전에 없는 단어는 "찾지 못함" 안내). → **시연은 미리 담긴 용어 hover로 진행.**
- **Gemini(SnowChat) 쿼터.** 문장 재구성/퀴즈 생성이 쿼터 초과 시 폴백으로 동작할 수 있습니다. 시연 전 `preflight.py`로 확인.
- **백엔드 창을 닫지 마세요.** 닫으면 서버가 죽습니다.
- **인터넷 없이도 됩니다.** 로컬 실행이라 시연장 wifi와 무관(사전/LLM 조회만 인터넷 필요).

## 5. 문제 해결 (Troubleshooting)

| 증상 | 원인 / 해결 |
|---|---|
| 브라우저에 빈 화면 | 서버가 아직 안 뜸 → 잠시 후 새로고침. `preflight.py`로 health 확인 |
| `start_demo.bat`가 바로 닫힘 | PowerShell 실행정책 → bat이 `-ExecutionPolicy Bypass`로 우회함. 그래도 안 되면 관리자 권한 명령창에서 `powershell -ExecutionPolicy Bypass -File demo\start_demo.ps1` |
| 퀴즈/점수 화면 500 에러 | 무Redis 폴백 버그는 수정됨(`redis.py`, `endpoints.py`). 최신 main인지 확인(`start_demo`가 자동 pull) |
| 포트 8000 이미 사용 중 | `stop_demo.bat` 먼저 실행 후 다시 시작 |
| 용어가 "찾지 못함"으로 뜸 | 사전 키 미설정(정상). 미리 담긴 용어(환각 등)로 시연 |
| 프론트 빌드 실패 | `apps/web`에서 `npm install` 다시 → `npm run build` |

## 6. 내부 메모 (기술)

- 실행: 폴더 루트에서 `python run.py` → `uvicorn backend.app.main:app` (0.0.0.0:8000)
- DB: `DATABASE_URL=sqlite+aiosqlite:///./literacy_care.db` (lifespan에서 테이블 자동 생성) · Redis 없으면 InMemory 폴백
- 프론트: `apps/web` Vite 빌드 → `3. Cognitive Care Backend/frontend_dist` 에 배치 → 백엔드가 `/` 로 SPA 서빙
- 키: `.env`(gitignore)에 `GEMINI_API_KEY` 등. 사전 키는 추후 시크릿으로 주입 예정
