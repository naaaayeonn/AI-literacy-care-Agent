대기 중

집중도 모니터 개발 하니스

사용법: ① 백엔드(localhost:8000)를 먼저 켠다 → ② 세션 시작 → ③ 모니터 열기(또는 Ctrl+Shift+F) → ④ 아래 시뮬 버튼을 누르거나 실제로 스크롤/창 전환하면 별도 창의 집중도 지수와 파라미터가 실시간으로 갱신된다.

이 페이지는 확장이 쓰는 tracker.js · debug_monitor.js · session_client.js를 그대로 로드한다. 즉 여기서 실시간으로 도는 흐름이 곧 확장에서 도는 흐름이다(확장은 여기에 크롬 팝업·storage 글루만 더한다).

인공지능 리터러시는 단순히 도구를 다루는 능력을 넘어, 생성형 모델이 만들어 내는 정보를 비판적으로 읽고 판단하는 역량을 뜻한다. 정보의 양이 폭발적으로 늘어난 오늘날, 글을 끝까지 집중해서 읽고 그 핵심을 스스로 정리하는 힘은 오히려 더 희소해지고 있다.

이 시스템은 사용자가 글을 읽는 과정 자체를 측정한다. 스크롤 속도, 창 전환, 무동작 구간 같은 읽기 행동 신호를 모아 집중도 지수를 계산하고, 집중이 흔들리는 순간을 감지하면 부드러운 개입으로 다시 글에 몰입하도록 돕는다. 요약을 대신 해 주는 것이 아니라, 읽는 힘 자체를 기르도록 설계되었다.

집중도 지수는 기본 100점에서 출발한다. 창을 벗어나는 이탈이 일어나면 큰 폭으로 감점되고, 글을 제대로 읽지 않고 빠르게 넘기는 스크롤도 감점 요인이 된다. 반대로 안정적으로 머무르며 읽으면 높은 점수가 유지된다. 이 점수는 같은 입력이면 같은 결과가 나오도록 재현 가능한 규칙으로 계산된다.

점수가 일정 구간 아래로 떨어지면 개입 단계가 올라간다. 75점 아래에서는 핵심 문장을 하이라이트하고, 50점 아래에서는 잠시 멈춰 다시 읽기를 권하며, 30점 아래에서는 방금 읽은 내용을 퀴즈로 확인하도록 유도한다. 이 모든 판단은 중앙 오케스트레이터가 하나의 세션 상태 위에서 조율한다.
